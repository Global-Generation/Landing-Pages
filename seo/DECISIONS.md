## 2026-04-27 — Не используем sat-* MCP, делаем gg-* отдельно

**Что решили:** Для `gg-search-console` и `gg-semrush` клонируем код из `~/.claude/mcp-servers/sat-*`, переименуем, новые env vars под global-generations.us.
**Почему:**

**Альтернативы:** SAT-серверы как опциональный профиль (env var SITE_URL — переопределить). Слишком хрупко.
**Последствия:**
- Lev делает шаги из `seo/references/MCP-SETUP.md` для нового OAuth flow.
- Дублирование кода — но MCP-серверы маленькие, окей.
**Кто принял:** Lev + agent.
**Когда пересматривать:** Если появятся ещё сайты — рефакторить в multi-tenant MCP с конфигом по domain.

**Кто принял:** Lev.
**Когда пересматривать:** После покупки SEMrush.

## 2026-04-27 — Подрезали toolkit до реального scope

**Что решили:** Удалили 5 slash-команд (`/seo-daily`, `/seo-weekly`, `/seo-keywords`, `/seo-diagnose`, `/seo-gaps`) и 8 documentation файлов (`PLAN.md`, `SYSTEM.md`, `KEYWORDS.md`, `COMPETITORS.md`, `CONTENT-PLAN.md`, `EDITORIAL-GUIDE.md`, `GLOSSARY.md`, `DAILY-LOG.md`). Осталось 5 команд и 5 main docs.

**Почему:**
- Сайт — single-page Tilda landing. SAT-уровень toolkit (10 команд + 13 docs) не соответствует.
- Половина документации была плейсхолдерами или придуманными правилами (EDITORIAL-GUIDE с brand voice который никто не подтверждал; COMPETITORS с угаданным DR; PLAN с aspirational roadmap без commitment).
- Daily routine для одностраничника overkill — нет ежедневных публикаций которые нужно мониторить.
- `/seo-keywords`, `/seo-diagnose`, `/seo-gaps` — все требуют SEMrush; ручной анализ через SEMrush UI проще пока команд мало.

**Альтернативы:**
- Оставить «на вырост» — risk: половина файлов deteriorates, документация стареет.
- Переписать каждый файл аккуратно — overhead больше чем ценность для текущего scope.

**Последствия:**
- 5 команд: `/seo-crawl`, `/seo-gsc`, `/seo-semrush`, `/seo-audit`, `/seo-report`.
- 5 main docs: README, STATUS, mapping, AUDIT, DECISIONS + trends.md + 4 reference файла.
- Удалённые файлы остались в git history — `git show e791baa:seo/PLAN.md` достанет если понадобится.

**Кто принял:** Lev (запрос «оцени что плохо») + agent (выполнение).
**Когда пересматривать:** Если появится блог / multi-page / контент-команда — вернуть `/seo-keywords`, `EDITORIAL-GUIDE`, etc. из истории.

## 2026-04-27 — H1 fix отменён: H1 уже существует в DOM

**Что решили:** Visually-hidden injection **не нужен**. Первый прогон `/seo-crawl` обнаружил что Tilda реально рендерит текстовый `<h1 class='tn-atom'>вы поступите <br>в вуз мечты</h1>` в DOM. Я раньше ошибся, утверждая что H1 отсутствует.
**Почему:**
- WebFetch summary первоначально вернул `H1: MISSING`, но это был артефакт обработки 4.7 MB HTML моделью — реальный grep по локальному файлу подтвердил: H1 есть, 1 штука, корректная иерархия (1 H1 / 17 H2 / 11 H3).
**Последствия:**
- Раздел `mapping.md → H1 fix` (visually-hidden injection) удалён.
- Новая проблема — H1 без primary keyword. Решается переписыванием существующего H1 текста (либо в Tilda Zero-block, либо sed-replace по уникальному `field='tn_text_1752245550869'`).
- В `mapping.md → H1 keyword fix` теперь конкретный sed под этот случай.
- Anti-pattern alert: **WebFetch на больших Tilda-HTML возвращает unreliable результат — всегда verify через локальный grep**.
**Кто принял:** agent (нашёл при тестовом прогоне), Lev узнает из этого decision log.
**Когда пересматривать:** Если решим переписать H1 либо в Tilda, либо через sed — соответствующий новый decision запись.

## 2026-04-27 — Блокируем AI-краулеров в robots.txt

**Что решили:** Оставляем существующую блокировку GPTBot, CCBot, anthropic-ai, Google-Extended, Bytespider, Amazonbot.
**Почему:** Не отдавать контент LLM-обучению (бизнес-решение Lev — контент должен быть конкурентным преимуществом).
**Альтернативы:** Allow всех — ради того чтобы попасть в LLM-ответы. Отвергли — низкая ценность референса в RU-LLM rezultаты + риск прямого копирования.
**Последствия:** Не появляемся в ChatGPT/Claude как референс. Но Googlebot не блокируется → обычный SEO работает.
**Кто принял:** Lev (унаследовано из Tilda-export, оставили).
**Когда пересматривать:** Если появятся данные что AI-search существенный канал трафика для education-ниши.
