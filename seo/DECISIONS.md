## 2026-04-27 — Title временно `РЕМАРКЕТИНГ`

**Что решили:** Поставили `<title>РЕМАРКЕТИНГ</title>` для теста deploy-пайплайна.
**Почему:** Нужно было проверить что цикл `edit → S3 sync → CloudFront invalidate` работает в свежей инфре после миграции домена.
**Альтернативы:** не делать тест (риск что в проде окажется поломка); делать на feature-странице (нет такой).
**Последствия:**
- Лендинг отдаёт «РЕМАРКЕТИНГ» в title и og до тех пор пока не вернём.
- Зафиксировано в `STATUS.md` как 🔥 Top Priority.
- DNS пропагация ещё не закончена → пока видно только на CloudFront-direct, на `global-generations.us` ещё SiteGround old version.
**Кто принял:** Lev + agent.
**Когда пересматривать:** до окончания DNS пропагации (2026-04-29). Lev назначит финальный текст.

## 2026-04-27 — Не используем sat-* MCP, делаем gg-* отдельно

**Что решили:** Для `gg-search-console` и `gg-semrush` клонируем код из `~/.claude/mcp-servers/sat-*`, переименуем, новые env vars под global-generations.us.
**Почему:**
- Sat-* серверы захардкодены под SAT property URL и SEO_DIR.
- Если перенастроить sat-* на global-generations — сломаем SAT SEO toolkit.
- Изоляция: один проект — один MCP.
**Альтернативы:** SAT-серверы как опциональный профиль (env var SITE_URL — переопределить). Слишком хрупко.
**Последствия:**
- Lev делает шаги из `seo/references/MCP-SETUP.md` для нового OAuth flow.
- Дублирование кода — но MCP-серверы маленькие, окей.
**Кто принял:** Lev + agent.
**Когда пересматривать:** Если появятся ещё сайты — рефакторить в multi-tenant MCP с конфигом по domain.

## 2026-04-27 — SEMrush deferred, не блокируем toolkit

**Что решили:** SEMrush API key Lev купит позже. Пока пишем команды без зависимости — `/seo-crawl`, `/seo-audit`, `/seo-weekly`, `/seo-report` работают без MCP.
**Почему:** Не блокировать запуск SEO-инфры на финансовое решение.
**Последствия:**
- 4 команды (`/seo-semrush`, `/seo-keywords`, `/seo-diagnose`, `/seo-gaps`) ⏳ не работают до покупки.
- Pre-emptive написали с правильными MCP-вызовами — после покупки сразу работают, переписывать не надо.
**Кто принял:** Lev.
**Когда пересматривать:** После покупки SEMrush.

## 2026-04-27 — Все SEO-файлы в `seo/`, а не `docs/` + `references/`

**Что решили:** Консолидировать `docs/seo*` и `references/*` в одну папку `seo/`. Зеркало структуры SAT.
**Почему:**
- Single entry point для будущих сессий и для других людей.
- Mirror SAT pattern — Lev уже привык.
- Легче grep по проекту: `seo/` ↔ `seo/references/` ↔ `seo/reports/`.
**Последствия:**
- Все 10 slash-команд обновлены — пути `docs/seo*` → `seo/*`.
- Старые `docs/`, `references/` директории удалены.
**Кто принял:** Lev (просьба) + agent (исполнение).
**Когда пересматривать:** Если появится non-SEO documentation — создать `docs/` отдельно для них.

## 2026-04-27 — Sitemap.xml оставляем как есть до первого `/seo-crawl`

**Что решили:** Не править руками 6 dead URL в sitemap. Первым прогоном `/seo-crawl` зафиксируем в отчёте, чисткой займёмся осознанно.
**Почему:**
- Возможно `/usa`, `/services`, etc. имели смысл в старой Tilda-многостраничной версии и Lev решит вернуть их.
- Лучше не делать irreversible cleanup из неполного контекста.
**Последствия:**
- В GSC будут 404 errors по 6 URL — терпим.
- Записано в Top Priorities, чтоб не забыть.
**Кто принял:** Lev + agent.
**Когда пересматривать:** После первого `/seo-crawl` отчёта (планово через 1-2 дня после OAuth).

## 2026-04-27 — Блокируем AI-краулеров в robots.txt

**Что решили:** Оставляем существующую блокировку GPTBot, CCBot, anthropic-ai, Google-Extended, Bytespider, Amazonbot.
**Почему:** Не отдавать контент LLM-обучению (бизнес-решение Lev — контент должен быть конкурентным преимуществом).
**Альтернативы:** Allow всех — ради того чтобы попасть в LLM-ответы. Отвергли — низкая ценность референса в RU-LLM rezultаты + риск прямого копирования.
**Последствия:** Не появляемся в ChatGPT/Claude как референс. Но Googlebot не блокируется → обычный SEO работает.
**Кто принял:** Lev (унаследовано из Tilda-export, оставили).
**Когда пересматривать:** Если появятся данные что AI-search существенный канал трафика для education-ниши.
