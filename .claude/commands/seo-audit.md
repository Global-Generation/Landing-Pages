# /seo-audit — Prioritized SEO action list

**Target file:** `docs/seo_mapping.md`
**Aux output:** `docs/seo-audit-YYYY-MM-DD.md`
**Argument:** `$ARGUMENTS` — пусто = все, `critical` = только 🔥, `all` = включая 🟢 defer.
**Requires:** ничего внешнего — только existing mapping.

---

## Step 1: Read mapping

Открыть `docs/seo_mapping.md`. Для каждой страницы извлечь:
- URL, priority (🔴🟡🟢), status, noindex
- Primary keyword + vol/KD + status
- Title/Description char count + ✓/⚠️
- H1 quality
- GSC данные (impressions, clicks, CTR, position, top queries)
- SEMrush position
- Diagnosis
- Open Issues & Actions

## Step 2: Keyword gate

Если primary keyword status = `🔍 Pending validation` или `📋 Pending approval`:
- Block keyword-aligned fixes (title rewrite, H1 rewrite, hero copy rewrite — всё что зависит от того _какой_ keyword)
- Technical fixes (длина title, schema, canonical, noindex) — продолжаем оценивать.

## Step 3: Issue scoring

Для каждой Open Issue/Action на странице — назначить:

**Impact:**
- High: vol ≥ 1000 ИЛИ страница 🔴 ИЛИ влияет на индексацию
- Medium: vol 200–999 ИЛИ страница 🟡
- Low: vol < 200 ИЛИ страница 🟢

**Effort:**
- Low: 15–30 мин (изменить meta-тег, sitemap, robots)
- Medium: 1–4 часа (переписать копирайт, добавить FAQ JSON-LD, перерендерить Tilda)
- High: дни (новые страницы, redesign блока, миграция)

**Priority matrix:**

| Impact \ Effort | Low | Medium | High |
|---|---|---|---|
| **High** | 🔥 Do First | 🔴 This sprint | 🟡 Plan |
| **Medium** | 🟡 This sprint | 🟢 Backlog | 🔵 Defer |
| **Low** | 🟢 Backlog | 🔵 Defer | ❌ Skip |

## Step 4: Update mapping

В каждой странице — Issues & Actions раздел:
- `[x]` для resolved
- `[ ]` для open
- Каждый пункт получает префикс матрицы: `[ ] 🔥 ...`

В `## SEO Health`:
- `### Top Priorities` — переписать сверху список 🔥 + 🔴 (max 7 штук)
- `### Open Issues` — переписать всё `[ ]` (sorted by priority)
- `### Status Board` — пересчитать (pages tracked / indexed / noindex / errors / last crawl)

`**Audit:** {YYYY-MM-DD}` в `## SEO Health`.

`docs/seo-audit-YYYY-MM-DD.md`:

Если `$ARGUMENTS = critical` — output только 🔥. Если `all` — все включая ❌ Skip.

## Notes

- Audit чисто читает mapping — никакие external API не дёргает. Самая дешёвая команда.
- Запускать после `/seo-crawl + /seo-gsc + /seo-semrush + /seo-diagnose` — чтобы issue-список был актуальный.
- Keyword gate важен — блокирует "переписать title под keyword X" если keyword X сам ещё не валидирован данными.
