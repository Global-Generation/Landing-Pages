# /seo-gsc — Pull GSC data into mapping

**Target file:** `docs/seo_mapping.md`
**Aux outputs:** `docs/seo-trends.md` (upsert строки за месяц), `docs/seo-gsc-YYYY-MM-DD.md` (dated отчёт)
**Argument:** `$ARGUMENTS` — `[/path]` для одной страницы, `[N days]` для окна (default 28).
**Requires:** MCP `gg-search-console` (см. `references/MCP-SETUP.md` для настройки).

Site URL в GSC: `https://global-generations.us/`.

---

## Step 1: Подтвердить доступ

```
mcp__gg-search-console__site_snapshot(site="https://global-generations.us/")
```

Если ошибка — стоп, попросить Lev проверить OAuth (см. `references/MCP-SETUP.md`).

## Step 2: Domain snapshot

```
mcp__gg-search-console__advanced_search_analytics(
  site="https://global-generations.us/",
  startDate=YYYY-MM-DD (today - N days),
  endDate=YYYY-MM-DD (today),
  dimensions=[]
)
```

Получить totals: impressions, clicks, CTR, avg pos.

## Step 3: Branded vs non-branded

Тот же endpoint с `dimensionFilterGroups` фильтрующим `query CONTAINS "global generation"` (и вариант без `s`).

```
brandedClicks = clicks где query содержит "global generation"
nonBrandedClicks = totalClicks - brandedClicks
```

## Step 4: Per-page

Если `$ARGUMENTS` задан — только эта страница. Иначе — все из `## Pages`.

Для каждой страницы:

```
mcp__gg-search-console__advanced_search_analytics(
  site="https://global-generations.us/",
  dimensions=["page"],
  dimensionFilterGroups=[{filters:[{dimension:"page", operator:"equals", expression:"https://global-generations.us{path}"}]}],
  startDate=..., endDate=...
)
```

Затем top queries для этой страницы:

```
mcp__gg-search-console__advanced_search_analytics(
  site="...",
  dimensions=["query"],
  dimensionFilterGroups=[{filters:[{dimension:"page", operator:"equals", expression:"..."}]}],
  rowLimit=10,
  startDate=..., endDate=...
)
```

## Step 5: Update mapping

В каждой странице `## Pages` обновить блок:

```
**GSC:** ({YYYY-MM-DD}, {N}d) Impressions: N · Clicks: N · CTR: X% · Avg pos: X.X · Top queries: [q1 (impr/clicks), q2 (...), q3 (...)]
```

В `## SEO Health → Domain Metrics` обновить:
- `Impressions (28d)`, `Clicks (28d)`, `Avg position`, `CTR` — текущие значения и Δ MoM (сравнить с прошлой строкой `seo-trends.md`).

В `## SEO Health` — `**GSC sync:** {YYYY-MM-DD}`.

**Intent mismatch flag**: для каждой страницы — если top GSC query НЕ содержит слов из её primary keyword → добавить пункт в `### Open Issues`: `[ ] /path: top GSC query "<q>" не совпадает с primary keyword "<kw>"`.

## Step 6: Trends

В `docs/seo-trends.md` апсерт строки за текущий месяц (анкер = последний день месяца):

```
| {YYYY-MM} | {impressions} | {clicks} | {ctr}% | {avg_pos} | {branded%} | GSC |
```

## Notes

- Default window: 28 дней (стандарт GSC).
- GSC возвращает данные с задержкой 2-3 дня — учитывай при weekly cadence.
- Если домен новый и данных мало — отчёт всё равно пишем, просто с малыми числами.
